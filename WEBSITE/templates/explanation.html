Bot Commands and Their Functions

/signup

Description: Registers your in-game name (IGN) and Discord user information in our database.

Usage: /signup IGN username

Details: When you use this command, the bot automatically records your Discord user ID and username. This registration is essential for us to add regear benefits to your account.

/payout

Description: Resets your silver balance to zero in our database after you've been paid in-game.

Usage: This command is used by COUNCIL members or higher.

Details: It marks your balance as paid, ensuring that your database entries are cleared, indicating that your in-game payout has been processed.

/Ball

Description: Displays your current silver balance from our database.

Usage: Use this command to check your own balance.

Details: The bot uses your Discord ID to automatically retrieve and display your silver balance.

/add_link

Description: Adds a specific amount of silver to users who appear on a designated killboard.

Usage: Managed by Regear Officers or higher.

Details: This command fetches data from a specific killboard on Albion Battles and credits the corresponding Smurfing Monkeys users in our database.

/add

Description: Allows Regear Officers to adjust a user's balance in the database.

Usage: Modify regear amounts or deduct specific sums.

Details: This command can be used to either add to or subtract from a user's total silver balance in the database.

/wb_countdown

Description: Sets up a two-day countdown for world boss respawn.

Usage: Schedule a world boss respawn timer.

Details: The countdown is automatically set to expire after 48 hours, matching the respawn interval for world bosses.

/remove_wb

Description: Removes a specific world boss countdown from the list.

Usage: Specify the countdown index to remove.

Details: The index starts from 1, and each countdown is listed as "Countdown 1," "Countdown 2," etc.

/content_in

Description: Sets a timer for Albion .4 resources, vortexes, or other in-game content.

Usage: Input the timer duration to track ongoing content timers.

Details: The command lists all active countdowns for various in-game activities.

/remove

Description: Removes a specific content countdown from the list.

Usage: Specify the countdown index to remove.

Details: Similar to /remove_wb, the index starts from 1 and each countdown is listed for easy identification.

/siphoned

Description: Checks a user's siphoned balance.

Usage: Display your siphoned balance from the database.

Details: This command does not use any Discord user information. The database is updated manually each day.

/total_siphoned

Description: Lists all users and their total siphoned balances.

Usage: View balances either alphabetically or in descending order.

Details: You can choose to display the list alphabetically or by the highest balance first, depending on user preferences.

/check_signup

Description: Verifies if a user is registered in the database.

Usage: Check your registration status.

Details: Use this command to confirm that your information is stored in the database.

Regear with Emoji Function

Description: Automatically adds regear amounts based on reactions in specified Discord channels.

Functionality:

  • When a user posts an image, emojis representing regear amounts (e.g., 1, 2, 3, etc.) are automatically added by the bot.
  • Regear Officers can click these emojis to add the specified regear amount to the user's balance.
  • If a regear officer presses a designated 'X' emoji, a thread is opened under that message.
  • Inside the thread, the regear officer can review the regear image and deny it based on the rules.
  • The regear officer selects a reason for denial from a list based on official rules.
  • The user who submitted the image for regear is pinged in the thread with the regears that were denied and the reasons provided.
  • Successful regears are confirmed similarly: 10,000 Regear added for user: "username". Regear added by: "Regear Officer's username".
  • The bot records the Discord user ID and username of both the image poster and the regear officer involved in the denial.

/setamount

Description: Sets the regear amount for the Emoji function.

Usage: This command is used by Regear Officers and higher to configure regear amounts.

Details: Establishes the regear values that are automatically assigned when the emoji function is used.

Important Notes

User Consent: If users do not sign up themselves, the bot will not collect or use any of their data. The bot first checks if it has data stored about a user, and if not, it does not gather or store any information about them.
